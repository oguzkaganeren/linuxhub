## 2024-03-06 - Add ARIA Labels to Modals and Action Buttons
**Learning:** Found that multiple modals and devices/printers panels in the app were using icon-only buttons (like the `Lucide-react` `<X />` component) for close or remove actions without an `aria-label`. This made them inaccessible to screen readers, causing them to read out "button" with no context. Using the existing translation function `t('close')` or plain text where appropriate is an easy way to fix this consistently.
**Action:** Always ensure that icon-only buttons (`<button>` tags with only an SVG/Icon child) have a descriptive `aria-label`. When working within a component that utilizes the `translations` setup, use `aria-label={t("close")}` to keep it localized.
