## 2024-05-18 - [Tauri Arbitrary Command Execution via Unrestricted Handlers]
**Vulnerability:** The Tauri application exposed two highly critical endpoints to the frontend: `run_shell_command_with_result` and `run_elevated_command`. These functions accepted raw strings from the frontend and passed them directly to `sh -c` and `/usr/bin/pkexec sh -c` respectively. This is a text-book, critical Command Injection vulnerability, granting the frontend arbitrary Remote Code Execution (RCE) and even arbitrary privilege escalation (Local Privilege Escalation, LPE).
**Learning:** This existed because the application developers attempted to create general-purpose command execution wrappers for the frontend rather than leveraging Tauri's secure, built-in `tauri-plugin-shell`. Tauri's security model explicitly requires commands to be whitelisted in `capabilities` or `tauri.conf.json` to prevent arbitrary execution if the frontend is compromised (e.g., via XSS).
**Prevention:** Never expose a generic "run command" endpoint to the frontend in a Tauri application. All necessary system interactions should be performed using Tauri's `tauri-plugin-shell` with strictly defined, scoped commands in the application's capabilities, or by creating specific, dedicated Rust commands that only perform a single, validated action without accepting arbitrary command strings.
